# Python packages
This is a list of preferred Python packages that should be used in org-wide contexts. You should use these instead of anything else for the same purpose.
## Production
These are packages to be used in production code and during development if needed.

- keyboard: Accessing the keyboard, e.g. check if a key is pressed
- mouse: Access the mouse
- yarl: Parse and handle URIs
- argparse: Parse and handle CLI (built in)
- pathlib: Parse and handle paths (built in)
## Dev
These packages should only be used during development.

- black: formatting
- pyright: type checking
- pytest: unit testing
