# All code
## Magic strings
AVOID using magic strings/numbers.
## NEVER write fallback code
NEVER write fallback code in case installed dependencies are missing or broken
NEVER write fallback code in case running in an older runtime
NEVER write fallback code in case running in a different OS
NEVER write fallback code when unsure of package type definitions
## Entrypoints
NEVER catch the error or rejection from an entrypoint function
NEVER check if an entrypoint is an entrypoint
## Functions
PREFER defining functions over repeating bits of code.
NEVER define functions with more than ~4 arguments
PREFER returning values to mutation or side-effects
## Declarations
PREFER properties instead of functions when possible
NEVER use properties if they cause side effects
