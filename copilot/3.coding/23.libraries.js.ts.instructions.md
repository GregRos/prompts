# Preferred libraries
This is a set of preferred JS/TS libraries for various tasks.

```yaml instructions
runtime: node@22
typescript: @latest
http:
	client: got
	# server: ask
	websocket: ws
validation: zod
cli: 
	parser: yargs
	color output: chalk
yaml:
	parsing: js-yaml
file:
	globbing: globby
	watching: chokidar
	access: fs/promises
execution:
	spawning: execa
images:
	processing: sharp
logging:
	framework: winston
	stringify object: yamprint
caching:
	base: keyv
	memory: @keyv/memcache

# dev packages:

	run multiple commands: npm-run-all
	formatter: prettier
	linter: eslint
	eslint preset: @gregros/eslint-config
	runner:
		typescript: tsx
	restart on change: nodemon
	testing:
		framework: jest
		ts loader: ts-jest
	bundling:
		pure js/ts: esbuild
		simple site: parcel
```

## WHEN user requests functionaltiy covered by preferred package:
1. Check current installation of packages for that functionality.
2. If none is installed, use preferred package.
3. Otherwise, use whatever is installed.
