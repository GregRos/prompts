## General
ALWAYS define strings and numbers in module level constants.
## Writing fallback code
NEVER write fallback for missing dependencies.
NEVER write fallback packages in an unexpected format.
NEVER write fallback code for lower runtime versions.
## Entrypoints
NEVER catch the error or rejection from an entrypoint function.
NEVER check if an entrypoint is an entrypoint.
## Functions
PREFER using a class WHEN:
- Two functions share an argument.
- Function takes more than ~4 arguments.
NEVER define functions with more than ~4 arguments.
PREFER splitting a function if it is longer than ~200 lines.
PREFER returning values to mutation or side-effects.
## Classes
PREFER immutable classes.
ALWAYS follow Single Source of Truth.
NEVER use `this` to pass data between methods.

## Names

PREFER names with ~4 or less words.
PREFER indicator properties for `foo` to be called `isFoo`
